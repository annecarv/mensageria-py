FLUXO DE FUNCIONALIDADE
=======================

Startup:
  1. pip install cryptography
  2. python3 init_certs.py          -> Gera RSA-2048
  3. python3 server.py               -> Servidor aguardando

Conexão Cliente A:
  4. python3 client.py "Alice"
     -> Conecta
     -> Handshake ECDHE+RSA+HKDF
     -> Deriva chaves
     -> Pronto para comunicar

Conexão Cliente B:
  5. python3 client.py "Bob"
     -> Conecta
     -> Handshake ECDHE+RSA+HKDF
     -> Deriva chaves
     -> Pronto para comunicar

Troca de Mensagens:
  6. Alice: > /msg <uuid_bob> Ola!
     -> Cifra com Key_c2s_Alice
     -> Envia ao servidor
  
  7. Servidor roteia:
     -> Decifra com Key_c2s_Alice
     -> Valida tag GCM
     -> Re-cifra com Key_s2c_Bob
     -> Envia para Bob
  
  8. Bob: Recebe
     -> Decifra com Key_s2c_Bob
     -> Valida tag GCM
     -> Exibe: "[Alice]: Ola!"

Encerramento:
  9. Alice/Bob: > /quit
     -> Fecha conexão
     -> Servidor limpa sessão

SEGURANÇA ALCANÇADA
===================

Confidencialidade:
  Sem chave, impossível ler
  AES-128: 2^128 chaves possíveis
  Força bruta: ~10^38 tentativas

Integridade:
  Tag GCM: 2^-128 probabilidade de falsificação
  Alteração: 1 bit muda a tag
  Detecção: 100% garantida

Autenticidade:
  RSA-2048: 2^2048 chaves privadas possíveis
  Assinatura: Apenas servidor consegue assinar
  Impede: MITM (man-in-the-middle)

Sigilo Perfeito:
  ECDHE independente de RSA
  Chaves: Não dependem de chave privada RSA
  Garantia: Sessões antigas seguras mesmo se RSA comprometida

Anti-Replay:
  seq_no: Contador monotônico por direção
  Validação: seq_no > seq_recv
  Impede: Replicação de mensagens antigas
